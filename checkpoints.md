# 開發檢核點 (Development Checkpoints)

以下是我們開發「Closed-List: 個人化封閉式清單應用程式」的主要檢核點，以及在每個點，當您執行 `pnpm dev` 後應該能看到和測試的內容：

---

### 檢核點 1：專案初始化與基礎佈局

*   **執行內容**：
    *   完成專案的建立 (`pnpm create astro`)。
    *   加入 React 和 Tailwind CSS 整合。
    *   建立一個基本的 Astro 頁面佈局 (`Layout.astro`) 和主頁 (`index.astro`)。
*   **檢測方式 (執行 `pnpm dev` 後)**：
    *   您應該會在瀏覽器中看到一個幾乎空白的頁面，上面可能只有一個標題，例如「Closed-List 應用程式」。
    *   您可以嘗試修改 Tailwind CSS 的類別（例如改變背景顏色 `bg-gray-800`），並立即在頁面上看到變化，以確認 Tailwind CSS 已正常運作。

---

### 檢核點 2：React 組件與狀態管理初次整合

*   **執行內容**：
    *   建立一個核心的 React 組件 (例如 `TaskManager.tsx`)。
    *   設定 Nanostores 並定義一個初步的、包含假資料的 `tasks` store。
    *   在 Astro 主頁中使用 `client:load` 指令渲染這個 React 組件。
*   **檢測方式 (執行 `pnpm dev` 後)**：
    *   您將會在頁面上看到由 React 渲染出來的內容，例如「我的任務清單」標題和一兩條寫死的假任務。
    *   這證明了 Astro 和 React 之間的「互動島嶼」模式已經成功建立，並且 React 組件能夠讀取 Nanostores 中的狀態。

---

### 檢核點 3：核心任務管理功能

*   **執行內容**：
    *   實作在 React 組件中新增任務到「今日封閉清單」的功能。
    *   實作點擊任務將其標記為「完成」（例如加上刪除線）的功能。
    *   將 Nanostores 的狀態與 `localStorage` 同步，實現資料持久化。
*   **檢測方式 (執行 `pnpm dev` 後)**：
    *   您現在應該可以：
        1.  在輸入框中輸入一個新任務，並將其加入到今日清單中。
        2.  點擊任務旁邊的複選框來完成它。
        3.  **重新整理頁面**，您會發現剛剛新增和完成的任務狀態依然存在。

---

### 檢核點 4：新任務與緊急任務處理

*   **執行內容**：
    *   實作「明天待辦」區塊，讓新任務可以被新增到這個獨立的、樣式較淡的區塊。
    *   實作「緊急插入」功能，讓任務可以直接被加入今日清單並附有特殊標記。
*   **檢測方式 (執行 `pnpm dev` 後)**：
    *   除了上一個檢核點的功能外，您現在還可以：
        1.  測試將一個任務加入到「明天待辦」區塊。
        2.  測試使用「緊急插入」功能，並看到該任務在今日清單中以不同樣式（如紅色文字）顯示。

---

### 檢核點 5：歷史紀錄與每日滾動邏輯

*   **執行內容**：
    *   建立歷史紀錄頁面 (`/history`)。
    *   實作每日結束時的狀態處理邏輯（自動滾動未完成任務等）。
*   **檢測方式 (執行 `pnpm dev` 後)**：
    *   執行 `pnpm dev`。
    *   您可以手動（或透過開發者工具）修改 `localStorage` 中的日期來模擬時間的流逝。
    *   您應該能看到：
        1.  昨天的未完成任務出現在今天清單的頂部。
        2.  昨天的「明天待辦」任務現在成為了今天清單的一部分。
        3.  您可以導航到 `/history` 頁面，並看到過去幾天的清單和完成狀態的唯讀檢視。
